Program ADC

#include <ModbusMaster.h>

#define DE_RE 4

ModbusMaster node;

void preTransmission() {
  digitalWrite(DE_RE, HIGH);
}

void postTransmission() {
  digitalWrite(DE_RE, LOW);
}

void setup() {
  Serial.begin(9600);
  Serial2.begin(9600);

  pinMode(DE_RE, OUTPUT);
  digitalWrite(DE_RE, LOW);

  node.begin(1, Serial2);   // Slave ID = 1
  node.preTransmission(preTransmission);
  node.postTransmission(postTransmission);

  Serial.println("=== Arduino MODBUS MASTER ADC READY ===");
}

void loop() {
  uint8_t result;

  // BACA HOLDING REGISTER (BUKAN INPUT REGISTER)
  result = node.readHoldingRegisters(0, 2);

  if (result == node.ku8MBSuccess) {
    uint16_t rawV1 = node.getResponseBuffer(0);
    uint16_t rawV2 = node.getResponseBuffer(1);

    float V1 = rawV1 * 5.0/512 ;   // V1 range 0–5V
    float V2 = rawV2 * 10.0/512;  // V2 range 0–10V

    Serial.print("V1 = ");
    Serial.print(V1, 2);
    Serial.print(" V | V2 = ");
    Serial.print(V2, 2);
    Serial.println(" V");
  } else {
    Serial.print("Modbus Error: ");
    Serial.println(result);
  }

  delay(1000);
}

1a. Jelaskan Alur (Flow) Program (Hafalkan 4 poin ini)

Inisialisasi: Di bagian setup, program melakukan inisialisasi komunikasi Serial (ke PC) dan Serial2 (ke Modul) dengan kecepatan 9600 baud. Program juga meng-inisialisasi pin DE/RE dan library Modbus untuk menargetkan Slave ID 1.

Request (Permintaan): Masuk ke loop, Arduino mengirim perintah (readHoldingRegisters) untuk meminta data dari modul.

Kontrol Arah (Pre/Post Transmission):

Sebelum kirim, fungsi preTransmission membuat pin DE HIGH (Mode Kirim).

Setelah kirim, fungsi postTransmission membuat pin DE LOW (Mode Terima/Dengar).

Respon & Konversi: Arduino menerima data mentah dari modul, lalu mengonversinya menjadi tegangan (Volt) menggunakan rumus dan menampilkannya di Serial Monitor.

1b. Mekanisme Protokol (Modbus RTU)
Cukup ingat 3 istilah ini:

Master-Slave: Arduino adalah Bos (Master) yang selalu bertanya duluan. Modul adalah Bawahan (Slave) yang cuma menjawab kalau ditanya.

Half-Duplex (Gantian): Komunikasi cuma satu jalur. Jadi harus gantian bicaranya (seperti Walkie-Talkie, tidak bisa ngomong barengan).

Data Hex & CRC: Data dikirim dalam bentuk kode angka (Hex), dan ada CRC di buntutnya untuk mengecek error (memastikan data tidak rusak di jalan).
